# Clawd Code 脑部系统 (Brain / World Model) 技术指南

## 1. 核心定义

`Brain` (位于 `src/brain/`) 是系统的认知中枢，其核心是 `WorldModel`。它赋予了 AI 代理“上帝视角”，使其能够理解代码库的全局拓扑、语义关系和设计模式。

## 2. 三位一体的索引机制

### 2.1 Trigram v2 索引
*   **用途**: 超高性能全文代码搜索。
*   **原理**: 基于 N-gram 的倒排索引。
*   **优势**: 毫秒级返回搜索结果，告别 `grep` 的线性扫描。

### 2.2 Tree-sitter 语法树
*   **用途**: 精确解析函数定义、类继承和导入关系。
*   **原理**: 使用高效的增量解析器生成 AST (抽象语法树)。
*   **集成**: 封装在 `rag/tree_sitter_syntax.py` 中。

### 2.3 RepoMap (代码图谱)
*   **用途**: 提供高压缩比的代码库摘要，作为 LLM 的核心上下文。
*   **原理**: 汲取自 Aider 算法，基于 Pagerank 原理对文件和符号的重要性进行排序，优先保留关键接口定义。

## 3. 语义检索 (RAG) 流程

1.  **查询解析**: 提取用户输入中的关键符号和技术意图。
2.  **多级索引查找**:
    *   一级：全局符号表匹配。
    *   二级：Trigram 相关性搜索。
    *   三级：依赖图 (Dependency Graph) 拓扑扩展。
3.  **结构化截断**: 在组装 Prompt 时，确保函数定义在语法边界处截断，绝不切断代码语义。
4.  **上下文晋升 (Promote)**: 将高频访问的代码片段或经验存储到 `EnterpriseLTM` (长期记忆) 中。

## 4. 认知新鲜度 (Freshness State)

为了防止 AI “刻舟求剑”，系统引入了新鲜度跟踪机制：
-   **文件指纹**: 监控源码变更。
-   **增量更新**: 仅在文件修改后重新索引受影响的部分，确保 World Model 实时反映代码现状。

## 5. 跨语言支持

World Model 设计为语言无关：
-   通过插件式 Parser 支持 Python, TypeScript, Go 等 10+ 语言。
-   统一的符号表示层，实现跨语言项目的语义联接。

## 6. 与 GoalX 的集成

`Brain` 模块同时负责管理 **Durable Surfaces (持久化表面)**：
-   **Memory Seed**: 任务启动时的初始记忆快照。
-   **Evidence Log**: 记录 AI 在理解代码时的推理过程和发现的潜在 Bug。
-   **Knowledge Retrieval**: 从历史运行中检索相似问题的解决方案。
