## 🔧 拆分方法论

### 通用拆分原则

1. **识别职责边界**: 找到代码中自然分离的点
2. **提取独立类**: 将相关功能封装为独立类
3. **委托模式**: 原类委托给新类，保持 API 兼容
4. **向后兼容**: 保留原有公共方法和静态方法
5. **测试验证**: 每次拆分后运行测试确保功能正常

### 拆分步骤

```python
# 1. 分析文件结构，识别职责边界
# 2. 创建新模块文件
# 3. 迁移相关代码到新文件
# 4. 在原文件中添加委托方法
# 5. 更新导入和依赖
# 6. 运行测试验证
# 7. 提交更改
```

## 📝 拆分最佳实践

### ✅ 推荐做法

1. **渐进式拆分**: 一次拆分一个文件，确保测试通过
2. **保留兼容层**: 原有 API 继续可用
3. **使用组合**: 优先组合而非继承
4. **懒加载**: 按需初始化，减少启动时间
5. **类型注解**: 保持完整的类型信息

### ❌ 避免做法

1. **大爆炸重构**: 一次修改太多文件
2. **破坏兼容**: 修改公共 API 签名
3. **过度设计**: 提前抽象不存在的场景
4. **忽略测试**: 拆分后不运行测试
5. **复制代码**: 在不同文件间重复逻辑
