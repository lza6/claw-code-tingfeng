# World Model (代码库感知核心)

World Model 是 Clawd Code 的"意识"层，负责在后台静默维护代码库的状态图，并为 Agent 提供精准的上下文支撑。

## 核心能力

- **统一管理**: 集成代码拓扑 (DependencyGraph) 和语义索引 (TextIndexer)。
- **设计模式检测**: 自动识别 Singleton, Observer, Factory, Strategy, Builder, Adapter 等常见设计模式。
- **关联预测**: 基于依赖关系（上下游）和语义相似度（Trigram 相似度）预测与当前任务相关的文件。
- **环境预取**: 在任务执行前提前加载并解析可能用到的文件内容或 AST，显著降低响应延迟。
- **内存友好**: 采用全异步轻量级逻辑，不依赖本地重型大模型。

## 工作原理

1. **初始化 (Scan)**: 系统启动时扫描项目文件，构建 `CodeGraph` (代码图谱)。
2. **AST 分析**: 使用 `PatternVisitor` 遍历 Python 文件的抽象语法树 (AST)，提取类定义、方法名和调用关系。
3. **语义索引**: 使用轻量级分词技术构建项目级文本索引，支持快速语义搜索。
4. **上下文指纹**: 为每个文件生成包含上游依赖、下游影响、语义关联及设计模式的"上下文指纹"。

## 使用场景

- **自动补全上下文**: 当 Agent 需要修改某个文件时，World Model 会自动推荐可能受影响的文件。
- **架构审查**: 通过图谱分析识别违反架构设计的依赖关系。
- **语义搜索**: 支持跨文件的功能逻辑定位，而非简单的字符串匹配。
